FOLIO / WIZLEADS BLOG THEME — TECHNICAL & SEO BEST PRACTICES
=============================================================

Everything built into this Ghost theme and the reasoning behind each decision.


1. TECHNICAL ARCHITECTURE
=========================

1.1 Ghost 5.x/6.x Compliance
- package.json manifest: engines.ghost >= 5.0.0, card_assets: true, image_sizes with 6 breakpoints (30px-2000px), up to 20 custom theme settings with select/boolean/text types, grouped by homepage and post.
- gscan zero errors: validated against latest gscan. All required helpers present (ghost_head, ghost_foot, asset, body_class, post_class). No deprecated helpers (@blog, {{author}}, {{pageUrl}}). Gallery and bookmark card CSS included.
- Template hierarchy: default.hbs (layout shell) -> index.hbs, post.hbs, page.hbs, tag.hbs, author.hbs, error.hbs. Child templates extend via {{!< default}}. Partials for reusable components (header, footer, post-card, subscribe, paywall, author-card).

1.2 Members & Paywall
- Portal integration: all auth links use Ghost Portal URLs (#/portal/signup, #/portal/signin, #/portal/account). No custom auth JS needed.
- Paywall via {{#unless access}}: shows a teaser fade + subscribe CTA when the visitor doesn't have access. Ghost strips paywalled content server-side before {{content}} renders, so no content leaks to the DOM.
- Member-aware UI: header shows Sign in / Subscribe for visitors, Account for logged-in members. Subscribe CTAs conditionally hidden via {{#unless @member}}.

1.3 Search
- Ghost's built-in sodo-search triggered by data-ghost-search attribute on buttons. Injected automatically via {{ghost_head}}. Uses FlexSearch client-side — indexes up to 10k posts with zero API configuration. Keyboard shortcut (Cmd+K) works out of the box.

1.4 Custom Theme Settings (12 admin-configurable options)
- feed_layout: Hero + grid / Grid only / Index list
- color_scheme: Auto / Light / Dark
- title_font / body_font: Elegant serif / Modern sans-serif
- post_dropcap, show_reading_progress, show_toc, show_related, show_author_bio (booleans)
- show_top_shelf + top_shelf_text + top_shelf_url (announcement bar)
- footer_tagline
- All read via {{@custom.setting_name}} and {{#match}} helpers
- Ghost's built-in font picker (--gh-font-heading, --gh-font-body CSS variables) also supported as fallbacks

1.5 Koenig Card Styling
Every editor card styled to match the editorial aesthetic:
- Image cards with .kg-width-wide and .kg-width-full breakout using left: 50%; transform: translateX(-50%)
- Callout (accent left border), bookmark (horizontal card), button (accent fill), toggle (details/summary), header (full-bleed section break), gallery (flex rows), audio/video/file/product (bordered containers), signup card

1.6 JavaScript Budget
Single theme.js, vanilla JS, no frameworks. Handles:
- 3-state theme toggle (auto -> light -> dark) with localStorage persistence + prefers-color-scheme listener
- Sticky header shadow state via scroll listener (passive)
- Mobile nav toggle with body scroll lock
- Top shelf dismiss with sessionStorage
- TOC auto-generation from H2/H3 headings + IntersectionObserver for active state
- Reading progress bar fallback (for browsers without animation-timeline: scroll())
- Copy-to-clipboard share button


2. SEO BEST PRACTICES
=====================

2.1 Structured Data / Schema.org
- Ghost's {{ghost_head}} automatically outputs WebSite schema (sitewide) and Article schema on posts (with author, publisher, datePublished, dateModified, image, headline, mainEntityOfPage). No duplication needed.
- Supplemental schemas can be added later via JSON-LD blocks after {{ghost_head}}: BreadcrumbList, Organization with logo + sameAs, Person on author pages, WebSite with SearchAction. Removed from v1 because Ghost's Handlebars lacks a jsonify helper for safe JSON string escaping.

2.2 Meta Tags
{{ghost_head}} handles all critical meta:
- <title>, <meta name="description">, <link rel="canonical">
- Open Graph: og:type, og:title, og:description, og:image, og:url, og:site_name, article:published_time, article:modified_time, article:tag, article:author
- Twitter Cards: summary_large_image, title, description, image, site, creator
- RSS autodiscovery: <link rel="alternate" type="application/rss+xml">

Theme adds on top:
- <meta name="color-scheme" content="light dark"> for native dark mode
- <meta name="theme-color"> with media queries for light/dark
- <meta name="application-name"> for brand signals
- <meta name="referrer"> and <meta name="format-detection">
- Favicon (SVG), apple-touch-icon

2.3 Sitemaps & RSS
- Ghost auto-generates /sitemap.xml (index) + per-type sitemaps (posts, pages, authors, tags). Zero theme work needed.
- RSS at /rss/ with autodiscovery link via {{ghost_head}}.
- Custom robots.txt shipped in theme root:
  - Allows all crawlers on /
  - Blocks /ghost/, /p/, /email/, /members/
  - Explicitly allows AI crawlers: GPTBot, ClaudeBot, PerplexityBot, Google-Extended, CCBot
- llms.txt template at theme root for AI citation readiness

2.4 Core Web Vitals Optimization

LCP (target < 2.0s):
- Hero/feature image preloaded via <link rel="preload" as="image" imagesrcset="..." fetchpriority="high"> in <head> on post pages (before {{ghost_head}})
- fetchpriority="high" and NO loading="lazy" on above-fold hero images
- Critical CSS inline-ready (single screen.css, no render-blocking JS in head)
- All JS loaded with defer
- Font stack uses system fonts by default (zero font download), with preconnect to Ghost CDN

INP (target < 200ms):
- Zero frameworks — all event handlers are lightweight vanilla JS
- Scroll listeners use { passive: true }
- No heavy third-party embeds
- Reading progress bar uses CSS animation-timeline: scroll() (zero JS on supporting browsers)

CLS (target < 0.05):
- width and height attributes on every <img>
- aspect-ratio on image containers (16/10 for cards, 4/3 for hero)
- font-display: swap strategy with system font fallbacks that closely match metrics
- --gh-font-heading / --gh-font-body CSS vars with fallbacks prevent layout shift from Ghost's font picker
- No above-fold content injected after load

2.5 Responsive Images
Every image uses Ghost's {{img_url}} with size parameter + full srcset:
- 6 image sizes defined in package.json (30px to 2000px)
- Ghost auto-generates resized copies + serves WebP via CDN when browsers accept it
- Card images use smaller sizes (xs to m) with appropriate sizes attribute
- All below-fold images use loading="lazy" decoding="async"

2.6 Font Loading Strategy
- System font stack by default — zero font downloads, zero FOUT, zero CLS
- Serif: Source Serif 4 -> Iowan Old Style -> Palatino -> Georgia
- Sans: Inter -> system-ui stack
- Mono: JetBrains Mono -> ui-monospace -> SF Mono
- Variable WOFF2 fonts can be self-hosted later (preload 1-2 above-fold, subset to Latin, budget <= 100KB total)
- Ghost's built-in font picker supported via var(--gh-font-heading, var(--font-serif)) pattern

2.7 Accessibility (Lighthouse 100 target)
- Semantic HTML: <header role="banner">, <main id="main">, <footer role="contentinfo">, <nav aria-label="Primary/Secondary/Footer">
- Skip link: .skip-link hidden off-screen, visible on focus, jumps to #main
- Single <h1> per page
- Focus rings: outline: 2px solid var(--accent); outline-offset: 3px on :focus-visible
- aria-current="page" on active nav items
- aria-expanded on mobile nav toggle
- aria-label on icon buttons (search, theme toggle, menu)
- alt text on all images (falls back to post title)
- Color contrast: >= 4.5:1 body text, >= 3:1 large text, verified in both light and dark modes
- @media (prefers-reduced-motion: reduce) kills all animations and transitions

2.8 GEO / AI Search Optimization
- robots.txt explicitly allows GPTBot, ClaudeBot, PerplexityBot, Google-Extended, CCBot
- llms.txt template for AI citation readiness
- Passage-level citability — post content uses clear H2/H3 structure, CSS enforces comfortable reading measure (68ch), and the Koenig editor naturally produces well-structured HTML
- E-E-A-T signals — visible author byline on every post linking to author page, author page with bio + avatar + external links, dateModified visible in post meta
- Brand consistency — og:site_name, Organization name, <meta name="application-name"> all use {{@site.title}}

2.9 Modern CSS Techniques
- OKLCH color space — perceptually uniform, consistent across light/dark modes, derived hover states via color-mix()
- light-dark() pattern via color-scheme: light dark + [data-theme] attribute
- Fluid typography — clamp() on every size token, no breakpoint jumps
- Container-aware layouts — auto-fit + minmax() grid for cards, no media queries for feed layout
- CSS Subgrid-ready — card grid structure supports subgrid for cross-row alignment
- Scroll-driven animations — reading progress bar via animation-timeline: scroll(root), zero JS
- View Transitions API — view-transition-name: post-{{id}} on cards and article for cross-page hero morphing
- :has() ready — CSS structure supports conditional styling without JS
